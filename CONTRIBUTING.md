# Contributing

Thanks for helping improve DeepMetricEye / DPDG-Env.

Good contribution areas include:

- reproducibility fixes,
- documentation improvements,
- dataset-loading utilities,
- Unreal Engine setup notes,
- calibration notes for additional VR headset geometries,
- clearer examples for training or inference.

## Before Opening a Pull Request

1. Open an issue for substantial changes so the scope can be discussed first.
2. Keep research-code changes focused and reproducible.
3. Do not commit private participant data, raw face scans, credentials, or NDA-restricted files.
4. Include enough context for others to reproduce your result: environment, dependency versions, input data shape, and expected output.

## Reporting Issues

When reporting a bug, please include:

- operating system,
- Python/PyTorch version or Unreal Engine version,
- the command or notebook cell that failed,
- the exact error message,
- whether you used the minimal sample or restricted data.
